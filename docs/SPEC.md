# 점 (Jeom) 언어 명세서

## Version 1.1.0

-----

## 1. 개요

**점(Jeom)** 은 유니코드 “점” 계열 문자 **16종**만으로 코드를 작성하는 난해 프로그래밍 언어입니다.  
공백(스페이스·탭·개행)은 토큰 구분자로만 사용됩니다.  
의미를 가지는 모든 토큰은 점 문자로만 구성됩니다.

- 파일 확장자: `.jeom`
- 실행 모델: 스택 기반 VM (Stack-based Virtual Machine)
- 엔진: `jeom_engine.js` (JS, UMD) / `jeom/` (Python)

## 3. 리터럴 인코딩

### 3.1 숫자 리터럴

형식: `•<비트열>•`  
비트: `.` = 0, `·` = 1  
비트열은 이진수 big-endian 표현.  
소수점: `‥` (U+2025) 로 정수부/소수부 경계 구분.

```
••        = 0
•·•       = 1   (binary 1)
•·.·•     = 5   (binary 101)
•·.·.•    = 10  (binary 1010)
•·‥.•     = 1.5 (1 + 0.5)
```

### 3.2 문자열 리터럴

형식: `●<바이트열>●`  
각 바이트는 8비트 이진수 (`.`=0, `·`=1).  
바이트 경계: `‧` (U+2027).  
인코딩: UTF-8.  
리터럴 내부의 공백(space/tab/LF)은 무시 (가독성용).

```
●.·......·‧.··.·..·●  =  "Hi"
  (H=72=01001000)  (i=105=01101001)
```

### 3.3 MAIN 마커

`•·` (U+2022 + U+00B7) 뒤에 공백/EOF/주석이 오는 경우에만 main 블록 시작으로 처리.  
`•·.·.•` 같은 숫자 리터럴과 구분됨.

## 7. 실행 모델 상세

### 7.1 값 타입

|타입  |설명                       |
|----|-------------------------|
|정수  |64비트 부동소수점으로 저장 (정수 범위 내)|
|실수  |64비트 부동소수점               |
|문자열 |UTF-8                    |
|배열  |동적 배열                    |
|딕셔너리|해시맵                      |
|함수  |클로저 포함 함수 객체             |
|null|없음                       |

### 7.2 Truthy 규칙

- `0`, `null`, `undefined`, 빈 문자열 `""`, 빈 배열 `[]` → falsy
- 그 외 → truthy

### 7.3 스택 연산 순서

```
A B ⋅   →   A + B   (A 먼저 push, B 나중 push)
```

pop 순서: B=첫 번째 pop (=b), A=두 번째 pop (=a)  
연산: a op b

### 7.4 함수 호출 규약

```
<인자1> <인자2> ... ˙˙˙ <함수이름>
```

인자는 순서대로 push. 함수 내부에서 선언 순서대로 바인딩.

## 8. WHILE 문법

```jeom
‥‥
⋮
  <조건 블록>   ◘ 스택에 1(계속) 또는 0(종료) push
⋮⋮
⋮
  <본문 블록>
⋮⋮
```

조건 블록을 매 반복마다 실행하고, 스택에서 팝한 값이 truthy면 본문 실행.

-----

## 9. TRY-CATCH-FINALLY 문법

```jeom
‥·
⋮  <try 블록>  ⋮⋮
‥··
⋮
  ◘ 에러 메시지가 스택에 push된 상태로 진입
  <catch 블록>
⋮⋮
‥·˙
⋮  <finally 블록>  ⋮⋮
```

CATCH와 FINALLY는 선택사항.

## 10. 변수 이름 규칙

변수 이름은 점 문자 시퀀스로 구성.  
명령 토큰과 동일한 시퀀스도 이름으로 사용 가능 (컨텍스트로 구분).  
예: `∘ ·· <값>` — 여기서 `··` 는 변수 이름 (PRINTLN 명령과 동일 시퀀스지만 VAR 컨텍스트에서는 이름).

